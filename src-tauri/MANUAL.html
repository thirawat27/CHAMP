C CHAMP Manual

Version 1.0.0

คู่มือใช้งาน CHAMP สำหรับเว็บ PHP ในเครื่อง

รัน Caddy, PHP-FPM, MySQL และ phpMyAdmin/Adminer ได้จากแอปเดียว พร้อมพอร์ตเริ่มต้นที่ไม่ชนกับ service ระบบและเก็บข้อมูลทั้งหมดไว้ในโฟลเดอร์ของผู้ใช้

Site localhost:8080
Database 127.0.0.1:3307
Default Tool phpMyAdmin 5.2.3

เมนูคู่มือ

เลือกหัวข้อที่ต้องการดูอย่างรวดเร็ว

  • 01Runtime ปัจจุบันเวอร์ชันที่แอปใช้ตอนนี้
  • 02ติดตั้ง Runtimeเลือกแพ็กเกจและ Install All
  • 03DashboardStart, Stop, Restart และ quick actions
  • 04URL และพอร์ตลิงก์เปิดเว็บและ database tool
  • 05Databaseข้อมูล login และตัวอย่าง PDO
  • 06Settingsปรับ port, project path และ tool
  • 07แก้ปัญหาแนวทางแก้ error ที่พบบ่อย

Runtime ปัจจุบัน

Caddy 2.11.2Web server หลัก เปิดเว็บผ่านพอร์ต 8080
PHP-FPM 8.5.xPHP runtime ค่าเริ่มต้น ใช้ official Windows NTS และ portable FPM สำหรับ Linux/macOS
MySQL 9.7.0Database service ค่าเริ่มต้น ใช้พอร์ต 3307
Adminer 5.4.2เครื่องมือจัดการฐานข้อมูลค่าเริ่มต้น โหลดเร็วกว่า และยังเลือก phpMyAdmin 5.2.3 ได้
CHAMP ใช้โฟลเดอร์ข้อมูลของผู้ใช้ เช่น %LOCALAPPDATA%\CHAMP บน Windows เพื่อหลีกเลี่ยงปัญหา permission และไม่ต้องรันด้วยสิทธิ์ administrator

ติดตั้งแอป

Windows

  1. ดาวน์โหลดตัวติดตั้งจาก GitHub Releases
  2. เปิด installer แล้วติดตั้งตามขั้นตอน
  3. ถ้า Windows SmartScreen แจ้งเตือน ให้เลือก More info แล้วกด Run anyway
  4. เปิด CHAMP จาก Start Menu หรือ shortcut

macOS และ Linux

ใช้ไฟล์ bundle ที่เผยแพร่ใน GitHub Releases แล้วเปิดใช้งานตามชนิดไฟล์ของระบบปฏิบัติการ เช่น .dmg บน macOS หรือ .AppImage บน Linux

ติดตั้ง Runtime ครั้งแรก

เมื่อเปิดแอปครั้งแรก CHAMP จะแสดง Setup Wizard เพื่อเลือกและติดตั้ง runtime ที่จำเป็น

  1. ตรวจรายการในหน้า Welcome และ Packages
  2. เลือก PHP version, Database package และ Database Tool ที่ต้องการ
  3. ค่าเริ่มต้นคือ PHP 8.5.x, MySQL 9.7.0 และ Adminer 5.4.2
  4. หน้า Review จะสรุปรายการก่อนติดตั้ง
  5. กด Install All เพื่อดาวน์โหลดและติดตั้งทุกอย่าง
Back Use Existing Keep & Install Install All
ถ้ามี runtime อยู่แล้ว ปุ่ม Use Existing จะใช้ของเดิม ส่วน Keep & Install จะเก็บของเดิมและติดตั้งรายการใหม่ตามที่เลือก

หน้า Dashboard

Dashboard ใช้ควบคุม service ทั้งหมดและเปิดเครื่องมือที่ใช้บ่อย

Service Cards

  • Caddy แสดงพอร์ตเว็บและ URL สำหรับเปิด site root
  • PHP-FPM แสดง FastCGI port สำหรับ Caddy ใช้ภายใน
  • MySQL แสดง database port และสถานะ service

สีปุ่มหลัก

Start / Start Stack Restart Stop

Quick Actions

  • Site เปิดเว็บที่ http://localhost:8080
  • phpMyAdmin หรือ Adminer เปิดตาม Database Tool ที่เลือกไว้
  • Projects เปิดโฟลเดอร์โปรเจกต์
  • Logs เปิดโฟลเดอร์ logs
  • Config เปิดโฟลเดอร์ config
  • GitHub เปิด repository ของ CHAMP

URL และพอร์ตค่าเริ่มต้น

รายการ ค่าเริ่มต้น วิธีใช้งาน
Site root http://localhost:8080 เปิดหน้าเว็บหลักจากโฟลเดอร์ projects
phpMyAdmin http://localhost:8080/phpmyadmin/ Database Tool ค่าเริ่มต้น
Adminer http://localhost:8080/adminer/ ใช้เมื่อเลือก Adminer ใน Package selection
PHP-FPM 127.0.0.1:9000 FastCGI ภายใน ไม่ใช่หน้าเว็บสำหรับเปิดใน browser
MySQL 127.0.0.1:3307 ใช้ต่อจาก PHP app หรือ database client
ใน Service Card ของแต่ละ service จะมี URL/Port ให้กดเปิดหรือคัดลอกได้ง่ายขึ้น ถ้าเปลี่ยน port ใน Settings URL จะเปลี่ยนตาม port ใหม่

โฟลเดอร์ Projects

วางไฟล์เว็บไว้ในโฟลเดอร์ projects แล้วเปิดผ่าน Caddy ที่พอร์ต 8080

projects/
  my-site/
    index.php
    assets/
      style.css
  another-site/
    index.php

ตัวอย่าง URL:

  • http://localhost:8080/my-site/
  • http://localhost:8080/another-site/

Settings

ใช้ Settings เพื่อปรับพอร์ต, โฟลเดอร์โปรเจกต์ และ runtime package ที่เลือกใช้งาน

  • Web Server Port ค่าเริ่มต้น 8080
  • PHP-FPM Port ค่าเริ่มต้น 9000
  • Database Port ค่าเริ่มต้น 3307
  • Project Root Directory สำหรับเก็บเว็บของคุณ
  • Database Tool เลือกได้ระหว่าง phpMyAdmin และ Adminer โดย phpMyAdmin เป็นค่าเริ่มต้น
  • PHP Versions ใช้ติดตั้งหรือสลับ PHP version ที่มีใน runtime config
Cancel Save
หลังเปลี่ยน port หรือ runtime package ควร Restart Stack เพื่อให้ config และ service ใช้ค่าใหม่ครบถ้วน

ตำแหน่งข้อมูล

ระบบ Base directory
Windows %LOCALAPPDATA%\CHAMP\
macOS ~/Library/Application Support/CHAMP/
Linux ~/.local/share/CHAMP/

ภายใน base directory จะมีโฟลเดอร์หลักเหล่านี้:

  • runtime เก็บ binary ที่ดาวน์โหลดมา
  • config เก็บ Caddyfile, php.ini และ config ของ database tool
  • mysql\data เก็บข้อมูลฐานข้อมูล
  • projects เก็บโปรเจกต์เว็บ
  • logs เก็บ log ของ service

แก้ปัญหาเบื้องต้น

Service start ไม่ขึ้น

  • ตรวจว่า port ไม่ถูกโปรแกรมอื่นใช้อยู่
  • ลองเปลี่ยน port ใน Settings แล้ว Restart Stack
  • เปิด Logs เพื่อดู error จาก service นั้น

phpMyAdmin ขึ้นว่าไม่ติดตั้ง หรือเปิดแล้วเป็นหน้า error

  • เข้า Setup Wizard แล้วติดตั้ง Database Tool อีกครั้ง
  • ตรวจว่าเลือก phpMyAdmin เป็น Database Tool ถ้าต้องการใช้ URL /phpmyadmin/
  • กด Restart Stack หลังติดตั้งหรือเปลี่ยนตัวเลือก

No input file specified.

  • มักเกิดจาก config ชี้ไปที่ tool หรือ project path ที่ไม่มีไฟล์จริง
  • ติดตั้ง Database Tool ให้ครบ แล้ว Restart Stack
  • ถ้ายังเกิดซ้ำ ให้ Reset Installation จาก debug/tools ของแอป แล้วติดตั้ง runtime ใหม่
