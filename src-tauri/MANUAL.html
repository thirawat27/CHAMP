CAMPP User Manual

Version 0.2.1 | Local Web Development Stack

Overview

CAMPP is a cross-platform local web development stack that bundles everything you need to run PHP websites locally:

  • Caddy 2.8.4 - Modern web server with automatic HTTPS
  • PHP-FPM 8.5.1 - Fast PHP runtime
  • Database - MySQL 8.4.0 on Windows/macOS, MariaDB 12.3.1 on Linux
  • phpMyAdmin 5.2.2 - Web-based database management

Installation

Windows

  1. Download the installer from GitHub Releases
  2. Run the installer (NSIS or MSI)
  3. Choose your project folder location during installation (default: %LOCALAPPDATA%\campp\projects)
  4. If Windows SmartScreen shows a warning, click More info → Run anyway
  5. Launch CAMPP from Start Menu or Desktop shortcut

macOS

  1. Download the .dmg file from GitHub Releases
  2. Open the DMG and drag CAMPP to Applications
  3. Launch CAMPP from Applications or Launchpad

Linux

  1. Download the .AppImage file from GitHub Releases
  2. Make the file executable: chmod +x CAMPP-*.AppImage
  3. Run the AppImage: ./CAMPP-*.AppImage
Note: The Windows SmartScreen warning appears because the app is not code-signed. This is normal and safe. The source code is publicly available on GitHub.
Windows Installation: You can select a custom project folder location during installation. On macOS and Linux, the project folder uses the default location but can be changed later in Settings.

First Run

On first launch, CAMPP will display a setup wizard that downloads the required runtime components (~500MB):

  1. Click Start Setup to begin downloading
  2. Wait for all components to download (this may take a few minutes)
  3. Click Get Started when complete

Dashboard

The main dashboard shows three service cards:

Service Controls

Each service card displays:

  • Status - Running, Stopped, or Error
  • Port - The port the service uses
  • Start/Stop/Restart buttons

Quick Actions (Top Right)

  • Open Site - Opens http://localhost:8080 in your browser
  • phpMyAdmin - Opens http://localhost:8080/phpmyadmin
  • Settings - Opens the settings panel

Default Ports

Service Port Access URL
Web Server (Caddy) 8080 http://localhost:8080
PHP-FPM 9000 Internal only (FastCGI)
Database (MySQL/MariaDB) 3307 localhost:3307
phpMyAdmin 8080 http://localhost:8080/phpmyadmin
Tip: These ports are chosen to avoid conflicts with system services. You can change them in Settings.
Note: CAMPP uses MySQL 8.4.0 on Windows and macOS, and MariaDB 12.3.1 on Linux. Both are MySQL-compatible and work identically for PHP applications.

Project Folder

Your website files should be placed in the project folder. The default locations are:

Windows

C:\Users\<YourUsername>\AppData\Local\campp\projects

macOS

~/Library/Application Support/campp/projects

Linux

~/.local/share/campp/projects
Project Folder Location:
  • Windows: Choose a custom location during installation.
  • macOS/Linux: Uses the default location above.

To quickly access this folder:

  1. Click Settings (gear icon)
  2. Click Open next to "Project Root Directory"

Example Project Structure

projects/
├── my-site/
│   ├── index.php
│   ├── about.php
│   └── assets/
│       └── style.css
└── another-site/
    └── index.php

Access your sites at:

  • http://localhost:8080/my-site/
  • http://localhost:8080/another-site/

Settings

Click the Settings button to configure:

Port Configuration

  • Web Server Port (default: 8080)
  • PHP-FPM Port (default: 9000)
  • Database Port (default: 3307)
Warning: Changing ports requires restarting all services to take effect.

Project Root Directory

Change where your project files are stored. Click Open to view the folder in Explorer.

Debug Menu

Press Ctrl+Shift+D to open the debug menu:

  • Open Runtime Folder - View downloaded binaries
  • View Download Folder - View downloaded ZIP files
  • Reset Installation - Delete all binaries and restart setup
  • Show First-Run Wizard - Re-run the initial setup

Troubleshooting

Service won't start

  • Check if the port is already in use by another application
  • Try changing the port in Settings
  • Check the error message displayed on the service card

phpMyAdmin shows errors

  • Make sure the database service (MySQL/MariaDB) is running first
  • Try restarting the database service

Reset everything

  1. Press Ctrl+Shift+D to open debug menu
  2. Click Reset Installation
  3. This will stop all services and delete runtime binaries
  4. Restart the app to re-download everything

Antivirus detects a threat

This is a false positive. The app is safe because:

  • Source code is open and available on GitHub
  • The warning appears because the app is not code-signed
  • Add an exception in your antivirus for the CAMPP folder

What Location
